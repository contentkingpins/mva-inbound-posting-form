üìã Overview

The MVA Lead API allows publishers to submit motor vehicle accident leads with comprehensive medical, legal, and compliance information. This enhanced data model provides law firms with the detailed information needed to properly evaluate and prioritize cases.

API Version: 2.1 (Enhanced MVA Support)
Protocol: HTTPS POST
Format: JSON
Encoding: UTF-8

üîê Authentication

All API requests require authentication using your publisher API key.

POST /api/leads Content-Type: application/json Authorization: Bearer YOUR_API_KEY X-Publisher-ID: YOUR_PUBLISHER_ID
Security Note: Never expose your API key in client-side code. All requests must be made from your server.

‚úÖ Compliance & Consent Fields

Compliance tracking and consent verification for regulatory requirements.

Field Name Type Required Description Example Values
trustedform_url string Optional TrustedForm certificate URL "https://cert.trustedform.com/abc123..."
consent string Optional User consent status "1" (consented), "0" (not consented)
Compliance Requirement: Ensure proper consent is obtained before lead submission. TrustedForm certification is highly recommended for compliance verification.

üìä Response Codes & Handling

Success Response

HTTP 200 OK
{ "status": "success", "lead_id": "lead_abc123def456", "message": "Lead submitted successfully", "timestamp": "2024-02-15T10:30:00Z" }

Error Responses

Status Code Error Type Description Action Required
400 Bad Request Missing required fields or invalid data format Check payload structure and required fields
401 Unauthorized Invalid or missing API key Verify authentication credentials
403 Forbidden Publisher account suspended or inactive Contact support
429 Rate Limited Too many requests - rate limit exceeded Implement exponential backoff and retry
500 Server Error Internal server error Retry request after delay
